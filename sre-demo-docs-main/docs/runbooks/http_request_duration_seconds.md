Алерт: `PodInfoHighLatency`
Severity: `WARN`
Impact: Замедление ответов сервиса (рост latency), ухудшение пользовательского опыта, возможны таймауты и деградация зависимых сервисов

1. Подтвердите проблему
	- Откройте дашборд или панель алерта и убедитесь, что рост latency устойчивый (не единичный всплеск)
	- Проверьте метрики:
		p95 / p99 latency (http_request_duration_seconds.*)
	- Выполните ручную проверку:
		curl -w "%{time_total}\n" -o /dev/null -s http://<service>

	- ожидаемо: время ответа в пределах нормы (< 1 сек)

2. Локализуйте источник задержек

	- Проверьте по слоям:
		Сеть / ingress / балансировщик
		нет ли задержек на уровне ingress
		не перегружен ли LB
		Сервис (PodInfo)
		все ли pod’ы в статусе Running
		нет ли рестартов
			kubectl -n demo get pods
		Ресурсы
		CPU / memory usage
		нет ли throttling или OOM
3. Проверьте логи сервиса

	- Посмотрите последние 10–15 минут:
		kubectl -n demo logs -l app=podinfo --tail=100

	- Ищите:
		медленные запросы
		таймауты
		ошибки зависимостей
		всплески нагрузки
4. Проверка зависимостей

	- Latency часто вызван не самим сервисом:
		База данных
		внешние API
		очереди

	- проверь:
		доступность
		время ответа
		ошибки
5. Примите минимальные меры для стабилизации
	- Если pod’ы перегружены:
		увеличить реплики:
		kubectl -n demo scale deployment podinfo --replicas=6
	- Если pod завис:
		перезапустить:
		kubectl -n demo delete pod <pod-name>
	- Если проблема в нагрузке:
		временно ограничить трафик (rate limit / ingress)
6. Проверьте улучшение
	- latency вернулась к норме (p95 < 1s)
	- алерт перешёл в RESOLVED
	- ручной curl показывает нормальное время
7. Эскалация (если не помогло)
	- Передайте:
		что проверили (pods, ресурсы, логи)
		что пробовали (scale / restart)
		текущий статус latency
8. Коммуникация
	- Пример сообщения:
		Алерт PodInfoHighLatency: наблюдается рост latency (p95 > 1s).
		Проверили pods и ресурсы, анализируем логи, возможна нагрузка или зависимость.
		Выполняем масштабирование/рестарт. Следующий апдейт через 5 минут.
